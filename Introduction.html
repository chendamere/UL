Introduction

In order to achieve machine thinking, or create a program that can deduce logical relation, "understand" fundamental definitions, and make decision base on observations in the data, there needs to be a sementically closed language equipped with as finite set of symbol manipulation operations to describe these paradigm.

For this purpose, the natural language has many weakness in describing logical relation, like the existence of paradox, which is still a major issue in any formal system, as demonstrated by Godel and many logicians after his famous incompleteness theorems. Readers are recommended to learn more about the this on the official stanford computer science website.

Therefore, a language for machine thinking needs to be constructed as a closed system that recognizes self-definitions and self-explanations. In fact, viewers browsing through the chapters will realize that the language does not rely on any natural language for its deduction and proofs. This sets a high learning curve for the language, since readers are expected to learn many components before reaching the later chapters on the construction of numbers and their basic operations.

Definitions, axioms, theorems, and proofs are the entire content of the first volume of the book. To simplify, the easier theorems will omit proofs. At the same time, theorems that can be proved using similar methods are no longer listed. Finally, in the more complex proof process, frequently occurring steps, especially the recovery process, will be omitted. For the arrangement of the chapters, some basic propositional concepts are defined to explain the basic principles of data structure and operation, at the same time, to prove a series of important theorems. Later chapters will define the concepts of number, addition, and multiplication, and demonstrate the fundamental laws of arithmetic. Finally, the last chapter defines paradox and proves that a paradox cannot lead to a contradiction. The concepts of infinity, abstraction, composition, and deeper mathematical concepts will be covered in the next volume.

This website will serve as a database and an assistance tool for learning about UL.

Data Structure

Universal language is designed to describe and manipulate data structure that qualifies as tree-like multidimensional structure with doubly linked list in each dimension.

Node

The basic element of data structure is node. A node consists of data value, unique node id, and links pointing to the next node, previous node, and child node. The empty node is represented with Ï•. There is exactly one empty node in one dimension. An empty node is used to identify the start and end of a one-dimensional loop. An empty node has no child node, so it points to itself
